User Requirements

1. User Journey Map

🏃 Before (Run Preparation):

Tom arrives at the park feeling tired and unmotivated. Sarah uses PC/mobile to set up virtual check-in points.

⚠️ During (Pain Points):

Tom finds it monotonous and忍不住想停下看手机确认路线；配速不稳定。

🏆 After (Run Completion):

Only sees cold numbers, lacks a sense of accomplishment; Sarah finds it difficult to track who completed the fun route.

2. Core Requirements List (Must-haves for Playfulness)

📍

Location-based Treasure Hunt Check-in

When runners pass through virtual geofences set by organizers, wearable devices will vibrate and collect "blind box" achievements.

👻

Virtual "Ghost" Running Companion

Based on historical data or organizer-set pace, visualize the distance in front/behind on wearable devices to add chasing fun.

🏆

Onsite Dynamic Leaderboard

Organizers can view real-time runner interaction status via mobile phone, facilitating on-site awards and interaction.
